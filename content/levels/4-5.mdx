你的全栈应用已经有了用户系统和数据库。但如果你想靠它赚钱，就需要支付功能。这一关教你用 AI 接入支付——比你想象的简单得多。

## 支付方案选择

| 方案 | 适合场景 | 费率 | 难度 |
|------|----------|------|------|
| **Stripe** | 海外用户/SaaS | 2.9% + $0.30 | 简单 |
| **LemonSqueezy** | 独立开发者/数字产品 | 5% + $0.50 | 最简单 |
| **支付宝当面付** | 国内个人 | 0.6% | 中等 |
| **微信支付** | 国内企业 | 0.6% | 需要企业资质 |

**推荐新手方案**：LemonSqueezy（不需要企业资质，全球收款，自动处理税务）

## 操作步骤

### 第一步：注册 LemonSqueezy

1. 访问 [lemonsqueezy.com](https://lemonsqueezy.com)
2. 注册账号（支持 GitHub 登录）
3. 创建一个 Store
4. 添加一个产品（比如"Pro 会员 - $9.9/月"）
5. 获取 API Key（Settings → API）

### 第二步：让 AI 帮你接入

对 AI 说：

```text
帮我给项目接入 LemonSqueezy 支付：

需求：
1. 安装 @lemonsqueezy/lemonsqueezy.js SDK
2. 创建 /api/checkout 接口，生成支付链接
3. 创建 /api/webhooks/lemonsqueezy 接口，接收支付回调
4. 在数据库中添加 subscriptions 表（user_id, plan, status, lemon_subscription_id）
5. 前端添加"升级 Pro"按钮，点击后跳转到支付页面
6. 支付成功后更新用户的 plan 状态
7. 添加一个简单的定价页面 /pricing

环境变量：
LEMONSQUEEZY_API_KEY=xxx
LEMONSQUEEZY_STORE_ID=xxx
LEMONSQUEEZY_WEBHOOK_SECRET=xxx

注意：
- Webhook 需要验证签名
- 处理 subscription_created 和 subscription_cancelled 事件
- 用 .env.local 存储密钥
```

### 第三步：测试支付流程

LemonSqueezy 提供测试模式：

```text
帮我配置 LemonSqueezy 测试模式：
1. 使用测试 API Key
2. 创建测试产品
3. 用测试信用卡号完成一次完整支付流程
4. 验证 webhook 回调正确更新了数据库
```

测试信用卡号：`4242 4242 4242 4242`（任意过期日期和 CVC）

### 第四步：添加付费墙

```text
帮我实现付费墙功能：
1. 创建一个 middleware 或 hook，检查用户是否是 Pro 会员
2. 某些页面/功能只有 Pro 用户才能访问
3. 非 Pro 用户看到"升级提示"弹窗
4. 在用户 dashboard 显示当前订阅状态
```

## 替代方案：Stripe

如果你的目标用户在海外，Stripe 是更专业的选择：

```text
帮我接入 Stripe 支付：
1. 安装 stripe 和 @stripe/stripe-js
2. 创建 /api/create-checkout-session 接口
3. 创建 /api/webhooks/stripe 接口
4. 实现 Stripe Checkout 流程（跳转到 Stripe 托管页面）
5. 处理 checkout.session.completed 事件
```

## 国内方案：支付宝

如果你的用户在国内：

```text
帮我接入支付宝当面付（个人可用）：
1. 注册支付宝开放平台，创建应用
2. 使用 alipay-sdk 包
3. 创建 /api/alipay/create 生成支付二维码
4. 创建 /api/alipay/notify 接收异步通知
5. 前端展示支付二维码，轮询支付状态
```

## 卡住了？

**Q: LemonSqueezy 注册需要什么？**
只需要邮箱和银行账户信息（用于收款）。个人就能注册，不需要企业资质。

**Q: Webhook 收不到回调？**
本地开发时需要用 ngrok 或 localtunnel 暴露本地端口。部署到 Vercel 后就不需要了。

**Q: 支付安全吗？**
你不会接触到用户的信用卡信息——所有敏感数据都由支付平台处理。你只需要处理支付结果通知。
