你的全栈应用已经有了用户系统和数据库。但如果你想靠它赚钱，就需要支付功能。这一关教你用 AI 接入支付——比你想象的简单得多。

## 选一个支付方案

先确定一个方案，然后跳到对应的注册流程：

| 方案 | 适合场景 | 费率 | 难度 |
|------|----------|------|------|
| **LemonSqueezy**（推荐新手） | 独立开发者/数字产品 | 5% + $0.50 | 最简单 |
| **Stripe** | 海外用户/SaaS | 2.9% + $0.30 | 简单 |
| **支付宝当面付** | 国内个人 | 0.6% | 中等 |

> 💡 **不知道选哪个？** 选 LemonSqueezy——不需要企业资质，全球收款，自动处理税务。等你真正有用户了再换也不迟。

---

**LemonSqueezy 注册方法：**

访问 [lemonsqueezy.com](https://lemonsqueezy.com)，注册账号，创建 Store → 添加产品（如"Pro 会员 - $9.9/月"）→ Settings → API 中获取 API Key。

接入时的 prompt 示例：

```text
帮我给项目接入 LemonSqueezy 支付：安装 @lemonsqueezy/lemonsqueezy.js SDK，创建 /api/checkout 接口，创建 /api/webhooks/lemonsqueezy Webhook 接口，在数据库添加 subscriptions 表，前端加"升级 Pro"按钮，支付成功后更新用户 plan 状态，添加 /pricing 定价页面。
环境变量：LEMONSQUEEZY_API_KEY / LEMONSQUEEZY_STORE_ID / LEMONSQUEEZY_WEBHOOK_SECRET
```

测试卡号：`4242 4242 4242 4242`（任意过期日期和 CVC）

---

**Stripe 注册方法：**

访问 [stripe.com](https://stripe.com) 注册，在 Dashboard → Developers → API Keys 获取密钥，在 Products 中创建订阅产品。

接入时的 prompt 示例：

```text
帮我接入 Stripe 支付：安装 stripe 和 @stripe/stripe-js，创建 /api/create-checkout-session 接口，创建 /api/webhooks/stripe 接口，实现 Stripe Checkout 流程，处理 checkout.session.completed 事件，前端添加定价页面和"升级"按钮。
环境变量：STRIPE_SECRET_KEY / STRIPE_PUBLISHABLE_KEY / STRIPE_WEBHOOK_SECRET
```

测试卡号同样是 `4242 4242 4242 4242`。

---

**支付宝当面付（国内用户）：**

访问 [open.alipay.com](https://open.alipay.com) 用支付宝账号登录，创建应用获取 AppID，配置公钥/私钥。

接入时的 prompt 示例：

```text
帮我接入支付宝当面付：使用 alipay-sdk 包，创建 /api/alipay/create 生成支付二维码，创建 /api/alipay/notify 接收异步通知，前端展示支付二维码并轮询状态。
```

## 操作步骤

### 第一步：注册支付平台并创建产品

根据你选择的方案（LemonSqueezy / Stripe / 支付宝），完成以下操作：

1. 注册账号（上方有各方案的注册入口）
2. 创建你的产品/价格（如"Pro 会员 $9.9/月"）
3. 获取 API Key，记录下来（待会要填到代码里）

有了"商品"和"钥匙"，才能继续下一步。

### 第二步：让 AI 接入支付后端

把上方对应方案的 prompt 示例复制给你的 AI 编程助手，让它帮你接入支付功能。

AI 写完代码后，把 `.env.local` 里的 `xxx` 替换成你第一步拿到的真实 Key，然后重启开发服务器。

### 第三步：测试支付流程（测试模式）

用测试信用卡号 `4242 4242 4242 4242` 走一遍完整支付流程：

1. 点击"升级"按钮，跳转到支付页
2. 填入测试卡号，完成支付
3. 支付完成后，去数据库查看 subscriptions 表是否多了一条记录

有记录 = 支付链路通了！

### 第四步：添加付费墙

对 AI 说：

```text
帮我实现付费墙功能：
1. 创建一个 middleware 或 hook，检查用户是否是 Pro 会员
2. 某些页面/功能只有 Pro 用户才能访问
3. 非 Pro 用户看到"升级提示"弹窗
4. 在用户 dashboard 显示当前订阅状态
```

做完这一步，你就有了一个完整的"免费试用 → 付费升级"的商业模式。

## 卡住了？

**Q: LemonSqueezy 注册需要什么？**
只需要邮箱和银行账户信息（用于收款）。个人就能注册，不需要企业资质。

**Q: Webhook 收不到回调？**
本地开发时需要用 ngrok 或 localtunnel 暴露本地端口。部署到 Vercel 后就不需要了。

**Q: 支付安全吗？**
你不会接触到用户的信用卡信息——所有敏感数据都由支付平台处理。你只需要处理支付结果通知。
