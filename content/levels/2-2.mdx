项目跑起来了，但现在只有一个默认页面。这一关把它改造成一个有结构的多页面网站。

组件化的核心思想是：把重复的东西抽出来，写一次，到处用。导航栏就是最典型的例子——第一章你在三个 HTML 文件里各写了一遍，改一处要改三遍。用 React 组件，导航栏只写一次，所有页面共享，改一次全改了。

## 操作步骤

### 第一步：搭建多页面骨架

对 AI 说：

```text
帮我把这个 React 项目改造成一个多页面的个人作品集网站：

页面结构：
- 首页（/）：个人介绍 + 精选作品预览
- 作品页（/works）：所有作品的卡片列表
- 关于页（/about）：详细的个人介绍

组件拆分：
- Header 组件：顶部导航栏，所有页面共享
- Footer 组件：底部页脚，所有页面共享
- 三个页面各自是独立的组件

路由：
- 用 React Router 实现页面切换
- 导航栏高亮当前所在页面
- 切换页面时不要整页刷新（SPA 单页应用）

内容先用占位文字就行，后面再填真实内容。
```

---

### 第二步：体验 SPA 的感觉

AI 完成后，在浏览器里点击导航栏的链接，注意观察：

- URL 变了（比如从 `/` 变成 `/works`）
- 但页面没有白屏/闪烁——只有内容区域更新了
- 顶部导航栏始终在，没有重新加载

这就是"单页应用"（SPA）——整个网站其实是一个页面，只是内容在切换。微信小程序、抖音 Web 版用的都是这个技术。

---

### 第三步：确认骨架结构合理

检查以下几点：

- 三个页面都能正常访问
- 导航栏在每个页面都显示，且当前页面的链接高亮
- 切换页面时没有整页刷新

如果有问题，告诉 AI 具体是哪里不对，让它修复。

## 通关条件

完成以下三步即可通关：

1. 让 AI 拆分组件并搭建路由
2. 三个页面切换正常（URL 变化，不整页刷新）
3. 确认骨架结构合理

## 卡住了？

- 路由不生效？让 AI 检查 React Router 是否正确安装和配置
- 导航高亮不对？告诉 AI："当前页面的导航链接没有高亮，帮我修一下"
- 页面切换时有白屏？让 AI 检查路由配置是否有问题
