首页做好了。现在做网站最核心的页面——**餐厅列表页**。用户来这里浏览所有餐厅，这一关你会学到卡片布局、标签筛选和 hover 交互。

## 操作步骤

### 第一步：生成餐厅列表页

继续在 `food-explorer` 项目里，对 AI 说：

```text
帮我创建一个 restaurants.html 文件，这是城市美食探索网站的餐厅列表页：

顶部导航栏：
- 左侧 Logo："🍜 城市美食"，白色文字
- 右侧导航链接：首页 / 餐厅列表 / 关于我们
- 深色背景 #1c1917，底部有一条细线分隔

筛选标签区域（导航栏下方）：
- 一排标签按钮：全部 / 川菜 / 粤菜 / 日料 / 西餐 / 火锅 / 小吃
- 默认"全部"高亮（橙色背景），其他标签是灰色边框
- 标签之间有间距，整体居中

餐厅卡片网格：
- 3 列网格布局（手机上变成 1 列）
- 每张卡片包含：
  - 顶部：一个 200px 高的图片区域（用渐变色块代替，每张用不同颜色）
  - 餐厅名称：加粗，16px
  - 标签：1-2 个小标签（如"川菜"、"人均 ¥80"）
  - 评分：⭐ 4.8 这样的格式，橙色
  - 一句话描述：灰色，14px，最多显示 2 行
  - 底部：一个"查看详情"链接
- 卡片有圆角、阴影，hover 时轻微上浮

请生成 8 张餐厅卡片，餐厅名称和描述用真实感的虚构数据（不要用"餐厅1"这种）。
```

### 第二步：检查并修复布局

在浏览器里打开 `restaurants.html`，检查：

- 卡片是否整齐排列成 3 列？
- hover 效果是否正常？
- 导航栏是否显示正确？

如果有问题，告诉 AI 具体哪里不对：

```text
卡片在我的屏幕上只显示了 2 列，帮我调整成 3 列，每列宽度相等。
```

### 第三步：让首页导航链接到列表页

回到 `index.html`，让 AI 把"开始探索"按钮链接到 `restaurants.html`：

```text
把 index.html 里的"开始探索"按钮的链接改成 restaurants.html，
同时在首页顶部加一个和 restaurants.html 一样风格的导航栏。
```

现在你有了两个页面，可以互相跳转了。

## 通关条件

完成餐厅列表页，要求：

1. 有导航栏（含 Logo 和链接）
2. 有筛选标签区域
3. 有至少 6 张餐厅卡片，卡片有图片区域、名称、评分、描述
4. 首页的"开始探索"按钮能跳转到列表页

## 卡住了？

- 卡片排列乱了？告诉 AI："帮我用 CSS Grid 实现 3 列等宽布局，列间距 24px"
- hover 效果不生效？让 AI 检查 CSS transition 属性是否正确
- 导航栏在手机上显示异常？告诉 AI："帮我让导航栏在手机上折叠成汉堡菜单"
