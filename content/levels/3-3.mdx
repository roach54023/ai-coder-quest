一个真正的网站需要让访客能联系你。这一关做一个联系表单——有完整的前端交互（验证、提示、动画），还可以选择接入真实的表单服务。

## 操作步骤

### 第一步：加入表单字段和验证

对 AI 说：

```text
帮我做一个联系表单页面（/contact）：

表单字段：
- 姓名（必填）
- 邮箱（必填，需要格式验证）
- 主题（选择框：合作咨询 / 技术交流 / 其他）
- 消息内容（多行文本框，必填，最少10字）

前端功能：
- 实时表单验证（输入时就提示错误，不是提交时才检查）
- 提交按钮在表单无效时变灰色且不可点击
- 点击提交后按钮显示 loading 旋转动画
- 提交成功后整个表单替换为一个感谢消息页面

样式：
- 和网站整体风格一致（深色主题）
- 输入框 focus 时有蓝色边框发光效果
- 错误提示用红色小字显示在对应字段下方
- 表单整体宽度限制在 500px，居中显示
```

### 第二步：确认提交功能正常

刷新浏览器，测试完整流程：

- 故意留空字段，看实时验证提示（红色小字）
- 填入错误邮箱格式，看报错提示
- 提交按钮在表单无效时变灰且不可点击
- 全部填对后点提交，看 loading 动画和成功感谢页面

如果验证不触发，对 AI 说："表单验证没有实时生效，帮我检查一下"

**进阶可选：接入真实的表单服务**

如果你想让表单真的能收到邮件，可以接入免费的 Formspree 服务：

1. 去 https://formspree.io 注册账号，创建一个表单，获取表单 ID
2. 对 AI 说："帮我把联系表单接入 Formspree，我的表单 ID 是 [你的ID]"
3. 测试提交后检查邮箱

> 这一步是可选的。即使不接入，只要前端交互完整也算通关。

## 卡住了？

- 表单验证不触发？对 AI 说："表单验证没有实时生效，帮我检查一下"
- 如果接入了 Formspree 但报 CORS 错误——确认你是从 `localhost` 访问而不是直接双击 HTML 文件
- 样式和网站其他页面不一致？对 AI 说："联系表单页面的风格和其他页面不统一，帮我调整"
