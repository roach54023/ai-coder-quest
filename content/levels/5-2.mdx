代码在 GitHub 上了，现在让它变成一个真正的网站——有 URL，全世界都能访问。

## 部署平台选择

| 平台 | 特点 | 免费额度 | 推荐场景 |
|------|------|---------|---------|
| **Vercel**（首推） | Next.js 官方平台，部署最简单 | 每月 100GB 带宽 | Next.js 项目 |
| **Netlify** | 老牌平台，稳定可靠 | 每月 100GB 带宽 | 静态站点、前端项目 |
| **Railway** | 支持数据库+后端一站式 | 每月 $5 免费额度 | 全栈项目 |
| **Zeabur** | 国内团队，对大陆友好 | 有免费计划 | 大陆用户首选备选 |

**推荐 Vercel**，因为我们用的是 Next.js，Vercel 是它的"亲爹"，兼容性最好。

## 方案一：Vercel 部署（推荐）

1. 打开 https://vercel.com 用 **GitHub 账号**登录
2. 点击 **"Add New → Project"**
3. 选择你刚才推送的 GitHub 仓库
4. 在 **Environment Variables** 中添加你的环境变量：
   - `NEXT_PUBLIC_SUPABASE_URL` = 你的 Supabase URL
   - `NEXT_PUBLIC_SUPABASE_ANON_KEY` = 你的 Supabase Anon Key
   - 其他 `.env.local` 中的变量（**不要**加 `.env.local` 文件本身）
5. 点击 **"Deploy"**
6. 等待 1-2 分钟，部署完成！
7. 你会得到一个 `https://xxx.vercel.app` 的地址

**避坑指南**

**坑1：环境变量没生效**

Vercel 的环境变量分三个环境：Production / Preview / Development。确保你的变量勾选了 **Production**。添加完变量后需要**重新部署**才会生效（点 Deployments → 最新一条 → Redeploy）。

**坑2：构建失败 — TypeScript 类型错误**

Vercel 构建比本地严格。本地 `npm run dev` 没问题不代表构建能过。部署前先在本地跑一次：

```bash
npm run build
```

如果报错，把错误信息发给 AI："帮我修这个构建错误：[粘贴错误]"

**坑3：API 路由 404**

确保你的 API 路由文件在 `app/api/` 目录下，且导出了正确的 HTTP 方法函数（`GET`、`POST` 等）。

**坑4：Supabase 连接失败**

在 Supabase 控制台 → Settings → API → 确认你的 Vercel 域名不在 CORS 限制中。如果用了 RLS，确保策略允许匿名访问公开数据。

**坑5：部署后页面白屏**

通常是客户端组件用了服务端 API。检查浏览器控制台（F12）的报错信息，发给 AI 帮你修。

## 方案二：Zeabur（大陆用户备选）

如果你访问 Vercel 速度很慢，可以试试 Zeabur：

1. 打开 https://zeabur.com 注册账号
2. 创建项目 → 添加服务 → 选择 Git → 连接 GitHub 仓库
3. 添加环境变量
4. 自动部署完成

Zeabur 的优势是服务器有亚洲节点，大陆访问速度更快。

---

## 方案三：Netlify

适合纯前端或静态站点：

1. 打开 https://netlify.com 用 GitHub 登录
2. "Add new site" → "Import an existing project" → 选仓库
3. Build command 填 `npm run build`，Publish directory 填 `.next`（或 `out`）
4. 添加环境变量 → Deploy

注意：Netlify 对 Next.js App Router 的支持不如 Vercel 完善，如果遇到问题优先用 Vercel。

---

## 方案四：Railway（全栈一站式）

如果你的项目需要后端服务或数据库：

1. 打开 https://railway.app 用 GitHub 登录
2. "New Project" → "Deploy from GitHub repo"
3. Railway 会自动检测 Next.js 项目并配置
4. 添加环境变量 → 自动部署

Railway 的优势是可以在同一个平台上跑数据库 + 后端 + 前端。

## 操作步骤

### 第一步：选择部署平台并导入仓库

参考上方的平台对比，选择适合你的平台（推荐 Vercel）。

注册并登录平台后，选择"导入 GitHub 仓库"，找到你上一关推送的仓库并选择它。

### 第二步：配置环境变量并点击部署

在平台的部署配置页面，将你 `.env.local` 文件中的所有环境变量逐一填入：
- `NEXT_PUBLIC_SUPABASE_URL`
- `NEXT_PUBLIC_SUPABASE_ANON_KEY`
- 其他密钥（注意不要把 `.env.local` 文件本身上传）

填好后点击 **Deploy**，等待 1-2 分钟构建完成。

如果构建失败，把构建日志最后几行红色错误复制给 AI："帮我修这个部署报错：[粘贴错误]"

### 第三步：验证线上可访问

部署成功后，你会得到一个公网地址（如 `https://xxx.vercel.app`）。

在浏览器打开这个地址，确认：
- 首页能正常加载
- 数据库连接正常（有数据显示）
- 手机浏览器也能打开

复制这个 URL，准备提交通关。
