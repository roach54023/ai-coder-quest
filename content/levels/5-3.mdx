你的网站现在有 AI 能力了。如果你想把某些功能做成付费的——比如"付费解锁完整简历"、"付费预约咨询"——就需要接入支付。

这一关用 Stripe 实现一个最简单的支付流程。

## 操作步骤

### 第一步：注册 Stripe 账号

1. 访问 [stripe.com](https://stripe.com)，注册账号
2. 注册后默认是测试模式，可以用测试卡号付款，不会真的扣钱
3. 在 Dashboard 左侧点击 "Developers" → "API keys"，复制 Publishable key 和 Secret key

---

### 第二步：创建付费产品

对 AI 说：

```text
帮我在网站上加一个付费功能：

场景：访客可以付费（¥29）解锁我的完整简历 PDF 下载

1. 安装 stripe：npm install stripe @stripe/stripe-js
2. 在 .env.local 里添加：
   STRIPE_SECRET_KEY=（你的 Secret key）
   NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY=（你的 Publishable key）
3. 在 Stripe Dashboard 里创建一个产品"完整简历"，价格 ¥29（或 $2.99 美元）
4. 创建 app/api/checkout/route.ts，生成 Stripe Checkout Session
5. 在关于页或首页加一个"解锁完整简历 ¥29"按钮，点击后跳转到 Stripe 支付页面
```

---

### 第三步：处理支付结果

对 AI 说：

```text
处理支付完成后的逻辑：

1. 创建支付成功页面 /payment/success：
   - 显示"支付成功！"和感谢文字
   - 显示简历 PDF 的下载按钮
   - 有返回首页的链接

2. 创建支付取消页面 /payment/cancel：
   - 显示"支付已取消"
   - 有重新支付的按钮

3. 在 Stripe Checkout Session 里设置 success_url 和 cancel_url 指向这两个页面

4. 用测试卡号 4242 4242 4242 4242（有效期任意未来日期，CVV 任意三位数）测试支付流程
```

## 通关条件

完成以下三步即可通关：

1. 点击"解锁完整简历"按钮，跳转到 Stripe 支付页面
2. 用测试卡号完成支付，跳转到成功页面
3. 成功页面有下载按钮（可以先链接到任意 PDF）

## 卡住了？

- 跳转到 Stripe 失败？检查 API Key 是否正确，确认用的是测试模式的 Key（以 `sk_test_` 开头）
- 支付后没有跳转？检查 success_url 是否是完整的 URL（包含 `https://`）
- 想在国内用？Stripe 在中国大陆需要境外银行卡，测试模式不受限制
