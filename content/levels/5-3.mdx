最后一关。你的网站已经上线了，但一个真正的产品还需要一些收尾工作。

## 操作步骤

### 第一步：SEO 优化完成

对 AI 说：

```text
帮我完善 SEO 优化：

1. 优化 SEO：每个页面有独立的 title 和 meta description
2. 添加 sitemap.xml
3. 添加 robots.txt
4. 确保环境变量没有泄露到前端
5. README.md 写清楚项目介绍和技术栈
```

完成后，用浏览器打开 `https://你的域名/sitemap.xml`，确认 sitemap 能正常访问。

### 第二步：社交分享卡片生效

继续对 AI 说：

```text
帮我添加社交分享卡片：

6. 添加网站 favicon（如果没有的话，用首字母生成一个简单的）
7. 添加 og:image（Open Graph 图片，分享到微信/微博时会显示的封面）
```

完成后，用 [opengraph.xyz](https://www.opengraph.xyz) 输入你的网站 URL，预览分享卡片的效果。

### 第三步：手机适配确认

推送最终代码到 GitHub，Vercel 会自动重新部署：

```text
帮我做最后的收尾检查：

8. 确保所有页面都能正常访问（没有 404）
9. 检查移动端适配，确保手机上所有功能正常
10. 在 Supabase 中设置 RLS（行级安全策略），保护数据
```

推送代码后等待 Vercel 自动完成部署，然后用手机浏览器访问你的网站，确认一切正常。

### 第四步：分享到社交平台

一切就绪，把链接发给一个朋友，让他们在手机上试试。

然后把你的作品分享出去——朋友圈、微博、即刻、Twitter/X，或者任何你活跃的地方。

粘贴你的线上网站 URL + GitHub 仓库地址，提交通关。

## 卡住了？

- RLS 策略不会写？对 AI 说："帮我给 posts 表设置 RLS，规则是：所有人可以读公开文章，只有作者可以编辑自己的文章"
- Lighthouse 分数低？让 AI 帮你优化，通常是图片和未使用的 JS 包导致的
- 如果 Vercel 自动部署没触发，检查 GitHub 仓库是否正确关联

## 恭喜通关！

回头看一下你现在手上有什么：前端界面、后端 API、数据库、用户系统、文件上传，部署在互联网上，谁都能访问。这就是一个完整的互联网产品。
