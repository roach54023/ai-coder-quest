把前面所有东西整合起来，做一个完整的全栈应用——一个内容管理系统（CMS）。

## 操作步骤

### 第一步：管理后台列表页

对 AI 说：

```text
帮我创建管理后台的文章列表页：

1. 用户系统：注册、登录、个人资料编辑
2. 文章列表：分页（每页10条）、搜索、按状态筛选（草稿/已发布）
3. Dashboard 统计卡片：总文章数、已发布数、草稿数

技术栈：Next.js + Supabase + TailwindCSS
```

完成后，确认文章列表能正常显示数据，分页和搜索功能可用。

### 第二步：添加/编辑功能

继续对 AI 说：

```text
帮我完善文章的增删改查功能：

4. 文章管理：创建、编辑、删除、切换发布/草稿状态
5. 富文本编辑器（用 Tiptap 或类似库）
6. 响应式设计，手机也能用
```

逐一测试：新建文章 → 用富文本编辑内容 → 保存 → 编辑 → 删除，确保流程顺畅。

### 第三步：图片上传功能

继续对 AI 说：

```text
帮我加入图片上传功能：

7. 图片上传（用 Supabase Storage）
8. 在富文本编辑器中可以插入上传的图片
```

在 Supabase 控制台创建一个 Storage bucket（如 `images`），设置公开访问策略，然后测试图片上传流程。

### 第四步：CRUD 完整流程走通

完成所有功能后，添加公开展示页面并做最终测试。对 AI 说：

```text
帮我加入公开的文章展示页面（不需要登录就能看），展示所有已发布的文章。
```

走完完整流程：登录 → 写文章+上传图片 → 发布 → 访问公开页面确认能看到 → 截图管理后台。

## 通关条件

截图管理后台 + 粘贴公开文章页面的 URL

## 卡住了？

- 这一关内容较多，建议分步完成：先做文章 CRUD → 再加富文本 → 最后加图片上传
- 如果 Tiptap 编辑器报错，让 AI 换用 `@tiptap/react` 最新版，或者换成更简单的 `react-quill`
- 图片上传失败？→ 确认 Supabase Storage 已创建 bucket，且设置了公开访问策略
- 如果功能太多一次搞不定，对 AI 说："我们一个功能一个功能来，先帮我做文章的增删改查"
