前三章你做的所有东西都在"前端"——用户直接看到和操作的部分。但它有一个大问题：**数据存不住。** 刷新页面，用户填的信息就没了；换一台电脑，什么都看不到。

"后端"就是解决这个问题的——它是网站背后的大脑和仓库，负责存数据、处理逻辑、对接各种服务。

---

## 关于你之前的项目

第二三章做的作品集网站先放一放——它已经是一个完整的前端作品了。这一章我们在**新项目**里学全栈能力（前端 + 后端 + 数据库），学完之后你有能力把作品集也升级成全栈应用。

为什么要新建项目？因为我们要换一个更适合全栈开发的框架（Next.js），它把前端和后端合在了一起，比之前分开管理更方便。

---

## 快速起步（推荐）

如果你想直接从一个配好认证和数据库的项目开始：

1. 打开 [fullstack-starter 模板](https://github.com/roach54023/fullstack-starter)
2. 点击 **"Use this template"** → Create a new repository
3. 克隆到本地，按 README 配置环境变量
4. `npm install && npm run dev`

> 这个模板已经包含了 Better Auth + Supabase 的完整配置，你可以直接在上面开发。

---

## 操作步骤

### 第一步：创建后端 API 接口

选择以下任一方式创建项目，然后创建你的第一个 API 接口：

**方式 A：使用模板（推荐）**

参考上方"快速起步"部分，克隆模板仓库并启动。

**方式 B：从零创建**

对 AI 说：

```text
帮我创建一个 Next.js 全栈项目（用 App Router）。
然后创建一个 API 路由 /api/hello，返回 JSON：
{ "message": "Hello from my API!", "timestamp": "当前时间" }
```

AI 会初始化项目并创建 API 路由。启动开发服务器后，在浏览器访问 `http://localhost:3000/api/hello`，看到 JSON 数据说明 API 创建成功。

### 第二步：前端 fetch 调通数据

API 能返回数据了，现在让前端页面也能取到它。对 AI 说：

```text
帮我在首页创建一个组件，用 fetch 请求 /api/hello 接口，
并把返回的 message 和 timestamp 显示在页面上。
```

刷新页面，看到 API 返回的内容显示在前端，说明前后端通了。

### 第三步：加 loading 骨架屏

数据在加载中时，页面不能一片空白。对 AI 说：

```text
帮我给 API 数据加载加上状态处理：
1. 加载时显示骨架屏（灰色占位块，有呼吸动画效果）
2. 加载成功时显示真实数据
3. 加载失败时显示错误提示，有一个重试按钮
```

### 第四步：截图 JSON 和前端

分别截图两样东西，准备提交通关：
1. 浏览器访问 `http://localhost:3000/api/hello` 看到的 JSON 数据
2. 前端页面显示从 API 获取的数据（带骨架屏/成功状态）

## 通关条件

粘贴你的 API 地址（如 `http://localhost:3000/api/hello`）

## 卡住了？

- 如果看到 404，检查文件路径是否正确：`app/api/hello/route.ts`
- Next.js API 路由必须导出 GET/POST 等函数
