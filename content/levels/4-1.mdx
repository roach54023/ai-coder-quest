到目前为止，你的网站是纯前端的——所有数据都写死在代码里，表单提交也只是模拟。

Next.js 有一个强大的功能：**API 路由**。你可以在同一个项目里写后端接口，不需要单独搭服务器。这一关来理解并用上它。

## 操作步骤

### 第一步：理解 API 路由

在 Next.js 里，`app/api/` 目录下的文件会自动变成 API 接口。

比如 `app/api/hello/route.ts` 就对应 `/api/hello` 这个接口。

对 AI 说：

```text
帮我创建一个最简单的 API 路由来理解它的工作方式：

创建 app/api/hello/route.ts，返回一个 JSON：
{
  "message": "Hello from API!",
  "time": "当前时间"
}

然后在首页加一个按钮"测试 API"，点击后调用这个接口，把返回的 message 显示在按钮旁边。
```

点击按钮，看看是否能显示 API 返回的内容。

---

### 第二步：让联系表单真正发邮件

上一章的联系表单只是模拟提交。现在让它真的能发邮件。

对 AI 说：

```text
帮我用 Resend 服务让联系表单真正发送邮件：

1. 安装 resend 包：npm install resend
2. 创建 app/api/contact/route.ts，接收表单数据（姓名、邮箱、主题、消息），用 Resend 发送邮件到我的邮箱
3. 更新联系表单，把提交请求改为调用 /api/contact 接口
4. 在 .env.local 里添加 RESEND_API_KEY 变量（我需要去 resend.com 注册获取）

注意：先用 console.log 打印接收到的数据，确认接口正常工作后再接入 Resend。
```

> **获取 Resend API Key**：访问 [resend.com](https://resend.com)，注册免费账号，在 Dashboard 里创建 API Key，复制到 `.env.local` 文件里。

---

### 第三步：测试接口

对 AI 说：

```text
帮我在联系页面加上接口测试功能：

- 提交表单后，如果接口返回错误，在表单上方显示具体的错误信息
- 在浏览器控制台打印接口的完整响应，方便调试
- 如果 RESEND_API_KEY 没有配置，接口返回友好的错误提示而不是崩溃
```

## 通关条件

完成以下三步即可通关：

1. `/api/hello` 接口可以访问，返回 JSON 数据
2. 联系表单调用真实的 API 路由（不再是 setTimeout 模拟）
3. 接口有错误处理，不会因为缺少配置而崩溃

## 卡住了？

- 接口返回 404？检查文件路径是否是 `app/api/xxx/route.ts`，函数名是否是 `GET` 或 `POST`
- 跨域报错？API 路由和前端在同一个 Next.js 项目里，不存在跨域问题，让 AI 检查请求 URL 是否正确
- Resend 发送失败？先确认 API Key 是否正确配置，再让 AI 查看错误信息
