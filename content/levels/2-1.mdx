第一章你用最原始的方式做出了一个网站——几个 HTML 文件，双击就能打开。这就像手写信件：能用，但效率低。

从这一关开始，我们换一套更强大的工具。**你不需要理解这些工具的原理——你只需要知道它们是干什么的，然后让 AI 帮你操作。**

---

## 为什么要升级？

你第一章的网站有几个痛点：

- **改一个导航栏，要改三个文件**——首页、关于、作品页的导航代码都是复制粘贴的
- **样式写在每个页面里**——想改个颜色得改三遍
- **没有预览服务器**——每次改完得手动刷新浏览器

现代开发工具解决的就是这些问题。类比一下：

| 第一章的方式 | 现在的方式 |
|---|---|
| 手写三封一样的信 | 写一个模板，自动印三份 |
| 改一个错别字改三遍 | 改一处，三份自动同步 |
| 写完打印出来才能看 | 写一个字屏幕就同步更新 |

---

## 新工具介绍（30秒了解即可）

你接下来会接触到这几个东西，它们各自负责一件事：

- **npm**：一个"包裹管理员"，帮你下载和管理别人写好的工具
- **Vite**：一个"快速启动器"，让你改了代码浏览器立刻更新（不用手动刷新）
- **React**：一个"组件积木系统"——把页面拆成一块块积木（导航、卡片、页脚），改一块全站生效

你不用学这些工具怎么用，让 AI 帮你操作就行。就像你不用学开车也能打车到目的地。

## 快速起步（推荐）

如果你想跳过初始化的技术细节，直接用我们准备好的模板：

1. 打开 [portfolio-starter 模板](https://github.com/roach54023/portfolio-starter)
2. 点击绿色的 **"Use this template"** 按钮
3. 给你的仓库起个名字，点击 Create
4. 对 AI 说：

```text
帮我把这个 GitHub 仓库克隆到桌面，然后安装依赖并启动开发服务器：
仓库地址：https://github.com/[你的用户名]/[你的仓库名].git
```

5. AI 会帮你执行命令，最后给你一个本地地址（通常是 `http://localhost:5173`）
6. 在浏览器打开，看到页面就成功了！

> 用模板起步完全算通关。后面几关会在这个基础上改造。

## 从零开始（想体验完整过程）

对 AI 说：

```text
帮我创建一个新的前端项目，用于做个人作品集网站：

要求：
- 使用 Vite + React + TypeScript 技术栈
- 在桌面创建一个叫 my-portfolio 的文件夹
- 初始化完成后自动启动开发服务器
- 启动后告诉我在浏览器打开哪个地址

注意：一步步来，每执行一个命令前告诉我这个命令是干什么的。
```

AI 会帮你完成所有初始化工作。你只需要在浏览器打开它给的地址（通常是 `http://localhost:5173`），看到页面就说明成功了。

## 操作步骤

### 第一步：AI 创建 Vite + React 项目

选择以下任一方式创建项目：

**方式 A：使用模板（推荐）**

参考上方"快速起步"部分，让 AI 帮你克隆模板仓库到桌面。

**方式 B：从零创建**

参考上方"从零开始"部分，让 AI 帮你初始化一个全新的 Vite + React 项目到桌面。

两种方式都可以通关，选你更顺手的那个。

### 第二步：npm install 安装依赖

项目文件下载到本地后，对 AI 说：

```text
帮我进入项目目录，运行 npm install 安装所有依赖
```

等待安装完成（终端不再有新输出就说明完成了）。如果下载很慢，对 AI 说："帮我换成国内镜像源再安装"。

### 第三步：npm run dev 跑起来

依赖安装好后，对 AI 说：

```text
帮我运行 npm run dev 启动开发服务器
```

AI 会告诉你浏览器打开哪个地址（通常是 `http://localhost:5173`）。在浏览器打开这个地址，看到页面就算通关！

## 通关条件

把 AI 给你的本地地址在浏览器打开，能看到页面就算通关。

## 卡住了？

- 如果 AI 执行命令时报错，把完整的错误信息复制给它，说"帮我解决这个报错"
- 如果报 `npm: command not found`——你的电脑还没装 Node.js。对 AI 说："帮我安装 Node.js，我是 Mac/Windows 系统"
- 如果 `npm install` 卡住不动很久——对 AI 说："npm 下载太慢了，帮我换成国内镜像源"
- 如果端口被占用（报 `EADDRINUSE`）——对 AI 说："端口被占用了，帮我换一个"
