你的 React 作品集已经很漂亮了，但它还只跑在你的电脑上。这一关用 Vercel 把它部署到互联网——而且连接 GitHub 之后，**以后每次 `git push`，网站自动更新**，不用再手动上传文件。

这就是"持续部署"——专业开发者的标准工作流。

---

## 操作步骤

### 第一步：把代码推送到 GitHub

如果你在 2-1 用了模板，代码已经在 GitHub 上了，可以跳到第二步。

如果是从零创建的项目，对 AI 说：

```text
帮我把这个 React 项目推送到 GitHub：
1. 初始化 git 仓库（如果还没有）
2. 创建 .gitignore 文件，忽略 node_modules 和 dist 目录
3. 提交所有文件
4. 告诉我怎么在 GitHub 上新建仓库并推送上去
```

按照 AI 的指引：
1. 在 [github.com](https://github.com) 新建一个仓库，命名为 `my-portfolio`（不要勾选 README）
2. 在终端执行 AI 给你的 git 命令

推送完成后，刷新 GitHub 仓库页面，能看到你的代码文件就说明成功了。

### 第二步：用 Vercel 连接 GitHub 仓库

1. 打开 [vercel.com](https://vercel.com)，点右上角 **Sign Up**，选 **Continue with GitHub** 用 GitHub 账号登录
2. 登录后点 **「Add New Project」**
3. 在仓库列表里找到 `my-portfolio`，点 **「Import」**
4. Vercel 会自动识别这是 Vite 项目，配置已经填好了，直接点 **「Deploy」**
5. 等 1-2 分钟，部署完成！

你会得到一个 `https://my-portfolio-xxx.vercel.app` 的网址。

### 第三步：验证自动部署

现在测试"推代码自动更新"的魔法：

对 AI 说：

```text
帮我在作品集首页的 Hero 区域加一行小字："🚀 持续部署测试 - 这行字是自动更新的"
然后帮我 git add、git commit 并 push 到 GitHub。
```

推送完成后，打开 Vercel 控制台，你会看到一个新的部署正在进行。等 1-2 分钟，刷新你的 Vercel 网址，新的文字出现了——**你没有手动做任何事，代码一推，网站自动更新**。

这就是现代开发的工作方式。

### 第四步：用手机验证并分享

用手机打开你的 Vercel 网址，确认：
- 页面正常显示
- 导航切换正常
- 响应式布局在手机上好看

把链接发给朋友看看！

## 通关条件

1. React 作品集成功部署到 Vercel，能通过公网链接访问
2. 完成一次代码修改 → git push → Vercel 自动更新的完整流程

## 常见问题

**Q: 部署后页面白屏？**
让 AI 帮你检查：`我的 Vite 项目部署到 Vercel 后白屏了，帮我排查原因`

**Q: 路由刷新 404？**
对 AI 说：`帮我在项目根目录创建 vercel.json，配置 SPA 路由重写规则`

**Q: 没有 GitHub 账号？**
去 [github.com](https://github.com) 免费注册一个，以后会经常用到。

**Q: Vercel 部署失败，报 Build Error？**
把 Vercel 的错误日志复制给 AI，说"帮我修复这个 Vercel 构建错误"。
