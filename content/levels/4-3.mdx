有了数据库，下一步是让不同用户有自己的数据。这一关实现完整的用户注册/登录系统。

## 操作步骤

### 第一步：做登录页面

在 Supabase 控制台中，进入 Authentication → Providers，确认 Email 已启用。

然后对 AI 说：

```text
帮我实现用户认证系统：

1. 使用 Supabase Auth
2. 注册页面（/register）：邮箱 + 密码
3. 登录页面（/login）：邮箱 + 密码
4. 登录后跳转到 /dashboard
```

完成后，在浏览器访问 `/register`，注册一个测试账号，完成邮件确认（如果有）后尝试登录。

### 第二步：导航栏登录态切换

继续对 AI 说：

```text
帮我完善登录状态的显示：

5. 导航栏显示当前用户邮箱，有退出按钮
6. 未登录用户访问 /dashboard 自动跳转到 /login
7. 加入 GitHub OAuth 登录（可选）
```

测试完整流程：注册 → 登录 → 看到导航栏显示邮箱 → 点退出 → 重新登录。

### 第三步：权限控制 /admin

让每个用户只能看到自己的数据。对 AI 说：

```text
帮我加入数据权限控制：

8. posts 表加一个 user_id 字段，关联到当前登录用户
9. 查询时只返回当前用户的文章
10. 在 Supabase 中设置 RLS（行级安全策略）：用户只能读写自己的数据
```

## 卡住了？

- 注册后没收到确认邮件？→ Supabase 免费版默认开启邮件确认，可以在 Authentication → Settings → 关闭 "Enable email confirmations" 来简化测试
- 如果登录后跳转不生效，检查是否正确监听了 `onAuthStateChange` 事件
- 如果 GitHub OAuth 配置失败，先跳过，用邮箱登录完成本关，OAuth 是可选的
- 如果报 `User already registered`，说明邮箱已注册过，直接去登录页
