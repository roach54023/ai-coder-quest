现在任何人都能访问你的 `/admin` 后台。这不安全。

这一关加上用户登录，让后台只有你能进。

## 操作步骤

### 第一步：开启 Supabase 认证

Supabase 内置了完整的用户认证系统，不需要自己写。

1. 打开 Supabase 项目，点击左侧 "Authentication"
2. 点击 "Providers"，确认 "Email" 已开启
3. 点击 "Users"，点击 "Invite user"，输入你的邮箱，发送邀请
4. 去邮箱点击邀请链接，设置密码

---

### 第二步：做登录页面

对 AI 说：

```text
帮我给网站加上用户登录功能：

1. 安装 @supabase/ssr：npm install @supabase/ssr
2. 创建登录页面 /admin/login：
   - 简洁的居中卡片布局
   - 邮箱和密码输入框
   - 登录按钮
   - 登录失败时显示错误提示
3. 登录成功后跳转到 /admin/works
4. 创建 app/api/auth/login/route.ts 处理登录请求

参考 Supabase 官方的 Next.js Auth 文档实现。
```

---

### 第三步：保护后台路由

对 AI 说：

```text
保护所有 /admin 路由，未登录时自动跳转到登录页：

1. 创建 middleware.ts，检查用户是否已登录
2. 访问 /admin/* 路径时，如果未登录，重定向到 /admin/login
3. 已登录时访问 /admin/login，重定向到 /admin/works
4. 在后台页面的右上角加上"退出登录"按钮，点击后清除登录状态并跳转到首页
```

退出登录后，尝试直接访问 `/admin/works`，确认会被重定向到登录页。

## 通关条件

完成以下三步即可通关：

1. 登录页面可以正常访问
2. 用你的邮箱和密码可以成功登录
3. 未登录时访问 `/admin/works` 会跳转到登录页

## 卡住了？

- 登录后还是跳回登录页？让 AI 检查 middleware.ts 里的 session 读取逻辑
- 邮件没收到？检查垃圾邮件文件夹，或者在 Supabase Authentication → Users 里直接修改密码
- 退出登录不生效？让 AI 确认退出时清除了 Supabase 的 session cookie
