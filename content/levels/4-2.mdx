API 能返回数据了，但数据从哪来？数据库。这一关接入 Supabase——一个免费的云数据库，自带管理界面。

> 💡 **Prompt 提示：** 这一关的 prompt 比较复杂（涉及数据库表结构、API 路由、前端页面），用好"分点列清楚"原则尤其重要——把数据结构、接口需求、页面需求分开描述，AI 才不会漏掉东西。

## 操作步骤

### 第一步：注册 Supabase

1. 打开 https://supabase.com 注册账号（用 GitHub 登录最快）
2. 创建一个新项目（选免费计划，区域选新加坡）
3. 等待项目初始化完成（约1分钟）
4. 进入项目后，在 **Settings → API** 中找到你的 **URL** 和 **Anon Key**，备用

### 第二步：创建数据表

对 AI 说：

```text
帮我接入 Supabase 数据库。

1. 安装 @supabase/supabase-js
2. 创建一个 lib/supabase.ts 配置文件
3. 在 Supabase 中创建一个 "posts" 表：
   - id (uuid, 主键, 自动生成)
   - title (text, 必填)
   - content (text)
   - created_at (timestamp, 默认当前时间)
4. 创建 API 路由：
   - GET /api/posts - 获取所有文章
   - POST /api/posts - 创建新文章

Supabase URL: [你的项目URL]
Supabase Anon Key: [你的anon key]
```

等 AI 完成后，检查 Supabase 控制台的 Table Editor，确认 posts 表已创建。

### 第三步：写入测试数据

创建一个简单的前端表单，添加几条测试数据。对 AI 说：

```text
帮我创建一个简单的前端页面，展示文章列表，并有一个表单可以添加新文章。
```

在页面上填写标题和内容，点击添加，然后刷新页面确认数据还在——数据持久化成功！

### 第四步：前端展示数据库数据

刷新页面后，你应该能看到从数据库读取的文章列表。截图这个页面（至少要有一条数据），准备提交通关。

## 卡住了？

- Supabase URL 和 Anon Key 在哪？→ 项目 Dashboard → Settings → API，第一个就是
- 如果页面报 `relation "posts" does not exist`，说明表还没创建。去 Supabase → Table Editor → New Table 手动建
- 如果数据写入成功但读取为空，检查 RLS（Row Level Security）是否开启了但没设置策略。最简单的办法：暂时关闭 RLS（Table Editor → 点表名 → RLS 开关）
- 如果报 `Invalid API key`，检查 `.env.local` 中的 key 是否复制完整（没有多余空格）
