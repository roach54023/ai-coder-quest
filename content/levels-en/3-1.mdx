Add a theme toggle so the interface can remember user preference and respond to system settings.

This page is a practical dark mode tutorial. You are not memorizing theory first; you are learning enough context to give better instructions, review AI's work, and ship something that behaves correctly.

## Why This Skill Matters

People looking for dark mode tutorial usually need more than a definition. They need a narrow workflow: what to ask AI, what to check in the browser, and what proves the result works.

In this level, dark mode tutorial stays tied to one outcome instead of drifting into unrelated tools or theory.

## What You Are Learning

### Theme is a user preference

Theme state is a simple example of user preference.

### Persistence makes preferences feel real

Local storage can preserve the choice after refresh.

### Accessibility is part of the feature

Accessible toggles need clear labels, contrast, and keyboard support.

## How to Work with AI in This Level

Treat the AI assistant like a fast junior developer that needs a clear brief and a reviewer. Give it the goal, the constraints, and the acceptance criteria. Then make it explain the files it changed before you move on.

A strong request usually includes:

- the user-facing outcome you want
- the pages, components, or files that should change
- the style or behavior constraints
- what should stay unchanged
- how you will verify the result

## Action Steps

### Step 1: Ask AI to add a theme toggle

Add a visible toggle with clear icon or label. The user should understand what changed immediately after clicking it.

Use this prompt as a starting point:

```text
Add a light/dark theme toggle to this portfolio. Support saved preference in localStorage, respect system preference if practical, and ensure all text remains readable in both themes.
```

After the assistant finishes, inspect the browser or terminal before continuing. The goal is to build the habit of checking real output instead of assuming the code is correct.

---

### Step 2: Support light mode, dark mode, and system preference if possible

Make both themes complete. Do not leave cards, borders, forms, or hover states unreadable in one mode.

After the assistant finishes, inspect the browser or terminal before continuing. The goal is to build the habit of checking real output instead of assuming the code is correct.

---

### Step 3: Refresh the page and confirm the chosen theme persists

Refresh the page after changing the theme. If the choice resets unexpectedly, ask AI to check local storage or system preference logic.

After the assistant finishes, inspect the browser or terminal before continuing. The goal is to build the habit of checking real output instead of assuming the code is correct.

## If You Get Stuck

- If AI makes a large change you do not understand, ask it to summarize the files changed and the reason for each change.
- If the page breaks, paste the exact browser console or terminal error into the assistant and ask for the smallest fix.
- If the result works locally but not after deployment, compare environment variables, build settings, and route paths.
